# Hardware Secret Treasures

## Flag

`HTB{m3M0ry_5cR4Mbl1Ng_4nd_1CG_423_n07_3n0u9h7!$#}`

## Files

- `embedded_software`: ARM ELF, not stripped
- `flash_memory_dump.bin`: 16 MiB flash contents
- `input_channel_trace.sal`: Saleae capture of the passcode line

## 1. Reverse the firmware

The firmware is an ARM binary with useful symbols:

- `main`
- `random_generator`
- `get_UniqieID`
- `get_secret`
- `W25Q128_init`

Important observations from `main`:

1. It reads **8 characters** from `/dev/ttyS0` at `0x9600` baud, then runs `strtoul(..., 10)`.
2. That decimal value is stored in the global `next_in_seq`.
3. It initializes the SPI flash and reads the flash chip unique ID with command `0x4b`.
4. The returned 8-byte unique ID is compared against the constant:

```text
d2 66 b4 21 83 51 30 2c
```

5. If the check succeeds, it prints **0x31 = 49** characters by repeatedly reading one byte from pseudo-random flash addresses.

The authentication step is device-specific and independent of the passcode. The passcode only seeds the PRNG used to choose the 49 output addresses.

## 2. Recover the PRNG

`random_generator` is:

```c
state = state * 0x41C64E6D + 0x8042A;
return state % 0xFFFFFF;
```

For each output character, the firmware builds a 24-bit flash address from **three** PRNG outputs:

```c
addr =
    (rand1 & 0xFF0000) |
    (rand2 & 0x00FF00) |
    (rand3 & 0x0000FF);
```

Then it reads `flash[addr]` and prints that byte.

## 3. Recover the passcode

The passcode is an 8-digit decimal string, so the seed space is only:

```text
0 .. 99,999,999
```

Because HTB flags start with `HTB{`, we can brute-force seeds and keep the one whose first four generated bytes match that prefix.

That returns the unique candidate:

```text
73261522
```

## 4. Extract the flag

This script reproduces the firmware logic directly from the flash dump:

```python
seed = 73261522
A = 0x41C64E6D
C = 0x8042A
MOD = 0xFFFFFF

with open("flash_memory_dump.bin", "rb") as f:
    flash = f.read()

def next_value(state):
    state = (state * A + C) & 0xFFFFFFFF
    return state, state % MOD

def next_addr(state):
    state, r1 = next_value(state)
    state, r2 = next_value(state)
    state, r3 = next_value(state)
    addr = (r1 & 0xFF0000) | (r2 & 0x00FF00) | (r3 & 0x0000FF)
    return state, addr

state = seed
out = []
for _ in range(0x31):
    state, addr = next_addr(state)
    out.append(flash[addr])

print(bytes(out).decode())
```

Output:

```text
HTB{m3M0ry_5cR4Mbl1Ng_4nd_1CG_423_n07_3n0u9h7!$#}
```

## Notes

The Saleae trace likely contains the same 8-digit passcode directly on the UART line, but once the firmware logic is known, brute-forcing the seed against the expected `HTB{` prefix is faster and simpler than decoding the capture format.
